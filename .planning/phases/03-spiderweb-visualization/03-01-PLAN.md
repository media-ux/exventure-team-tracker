  <name>Task 2: Create useGraphData hook for hierarchical data</name>
  <files>src/hooks/useGraphData.ts</files>
  <action>
Create custom hook that fetches hierarchical data from Supabase and transforms it into graph nodes and links format.

**File:** `src/hooks/useGraphData.ts`

**Interface requirements:**
```typescript
interface GraphNode {
  id: string
  name: string
  level: 'company' | 'project' | 'sub-unit' | 'task'
  status?: 'done' | 'in_progress' | 'backlog' | 'blocked'
  projectId?: string
  assignees?: string[]  // First names only
  x?: number
  y?: number
}

interface GraphLink {
  source: string
  target: string
}

export function useGraphData(hierarchyLevel: string, parentId: string | null)
```

**Implementation:**
1. Use existing supabase client from `src/lib/supabase.ts`
2. Fetch data based on hierarchyLevel:
   - 'company': Fetch company node + all projects
   - 'project': Fetch project node + all sub_units for that project
   - 'sub-unit': Fetch sub_unit node + all tasks for that sub_unit
3. For each level, join with team_members to get assignee first names
4. Transform into nodes array (each entity becomes a node) and links array (parent→child relationships)
5. Include loading and error states
6. Use type-only imports for Database types (follow pattern from existing components)

**Query pattern (example for project level):**
```typescript
const { data: projects } = await supabase
  .from('projects')
  .select(`
    *,
    sub_units (
      *,
      tasks (
        assigned_to,
        team_members:assigned_to (name)
      )
    )
  `)
  .eq('company_id', parentId)
```

**Data transformation:**
- Create parent node (company/project/sub-unit)
- Create child nodes (projects/sub-units/tasks)
- Create links connecting parent to each child
- Extract first names from team_members.name using split(' ')[0]

Follow existing hook patterns from `useRealtimeSubscription.ts` (useState, useEffect, cleanup).

Create React component that renders force-directed graph using ForceGraph2D with static layout configuration per user requirements.

**File:** `src/components/SpiderwebGraph.tsx`

**Props interface:**
```typescript
interface SpiderwebGraphProps {
  hierarchyLevel: 'company' | 'project' | 'sub-unit' | 'task'
  parentId: string | null
}
```

**Implementation:**
1. Import ForceGraph2D from 'react-force-graph-2d'
2. Use useGraphData hook to fetch data
3. Use useRef for graph instance (for later zoom control)
4. Configure ForceGraph2D with static layout settings from research:
   - `warmupTicks={100}` - pre-compute layout before render
   - `cooldownTime={2000}` - freeze after 2 seconds
   - `d3AlphaDecay={0.05}` - faster settling
   - `d3VelocityDecay={0.4}` - smooth deceleration
   - `enableNodeDrag={false}` - no dragging per user requirement
   - `enablePanInteraction={true}` - allow pan
   - `enableZoomInteraction={true}` - allow zoom
5. Configure expansive layout forces via d3Force prop:
   - charge force: strength -400 (strong repulsion for sparse feel)
   - link force: distance 150 (long links)
6. Set dimensions to window.innerWidth and window.innerHeight
7. Use inline styles (following project pattern from existing components)
8. Show loading skeleton during data fetch (use react-loading-skeleton from package.json)
9. Show error message if data fetch fails

**Basic node/link styling (advanced styling in Plan 03-02):**
- nodeColor: gray for now (will be project-based in 03-02)
- nodeVal: return 10 (uniform size for now, will vary by level in 03-02)
- linkColor: '#cccccc' (light gray, per research)
- linkWidth: 1 (thin lines per user requirement)

Do NOT implement custom node rendering, tooltips, or click handlers yet - those are in Plans 03-02 and 03-03.

Export as named export: `export function SpiderwebGraph({ hierarchyLevel, parentId }: SpiderwebGraphProps)`
