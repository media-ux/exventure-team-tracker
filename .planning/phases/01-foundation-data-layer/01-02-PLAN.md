<task type="auto">
  <name>Task 2: Create Row-Level Security policies for team-based access control</name>
  <files>
    supabase/migrations/20260228000000_initial_schema.sql
  </files>
  <action>
    Append RLS policies to the same migration file created in Task 1. Implement policies from 01-RESEARCH.md (Section: RLS Policies for Team-Based Access).

    For each table, create policies following this pattern:

    **team_members:**
    - SELECT: Users can view team members in their company (check company_id matches user's company)
    - INSERT: Users can add team members to their company (admin role check if needed later)

    **projects:**
    - SELECT: Users can view projects in their company
    - INSERT: Users can create projects in their company
    - UPDATE: Users can update projects in their company
    - DELETE: Users can delete projects they created

    **sub_units:**
    - SELECT: Users can view sub-units for projects in their company
    - INSERT: Users can create sub-units under projects in their company

    **tasks:**
    - SELECT: Users can view tasks in their company's projects
    - INSERT: Users can create tasks under sub-units in their company
    - UPDATE: Users can update tasks they created OR are assigned to
    - DELETE: Users can delete tasks they created

    CRITICAL: Wrap auth.uid() in SELECT for performance: `WHERE user_id = (SELECT auth.uid())`

    Use multi-level JOINs for hierarchical checks:
    ```sql
    -- Example: Task access requires checking company through sub_unit -> project -> company
    USING (
      sub_unit_id IN (
        SELECT su.id FROM sub_units su
        INNER JOIN projects p ON su.project_id = p.id
        INNER JOIN team_members tm ON p.company_id = tm.company_id
        WHERE tm.user_id = (SELECT auth.uid())
      )
    )
    ```

    Add TO authenticated to all policies (no public access).
  </action>
  <verify>
    Count policies in migration file: `grep -c "CREATE POLICY" supabase/migrations/20260228000000_initial_schema.sql` (should be 15+).
    Verify all policies use `TO authenticated` and check `auth.uid()`.
    Verify no policies use direct `auth.uid()` without SELECT wrapper.
  </verify>
  <done>
    All tables have SELECT, INSERT, UPDATE, DELETE policies enforcing team-based access, policies use (SELECT auth.uid()) for performance, no `TO public` policies exist.
  </done>
</task>
