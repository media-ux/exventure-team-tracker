  <action>
    Create src/lib/ directory and implement Supabase client singleton following Pattern 1 from 01-RESEARCH.md (Section: Architecture Patterns).

    Create src/lib/supabase.ts:
    ```typescript
    import { createClient } from '@supabase/supabase-js'

    const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
    const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

    if (!supabaseUrl || !supabaseAnonKey) {
      throw new Error('Missing Supabase environment variables. Check .env.local for VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY')
    }

    export const supabase = createClient(supabaseUrl, supabaseAnonKey)
    ```

    Create .env.local file with placeholder values (user will replace with actual Supabase project credentials):
    ```
    VITE_SUPABASE_URL=https://your-project.supabase.co
    VITE_SUPABASE_ANON_KEY=your-anon-key-here
    ```

    Create .env.example (committed to git as template):
    ```
    VITE_SUPABASE_URL=
    VITE_SUPABASE_ANON_KEY=
    ```

    IMPORTANT: Use `VITE_` prefix for client-side variables (Vite exposes them to browser). DO NOT use `VITE_` prefix for secrets like service role keys.
  </action>
  <verify>
