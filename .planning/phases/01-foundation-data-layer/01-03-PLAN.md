    Update src/lib/supabase.ts to import and use generated Database types:
    ```typescript
    import { createClient } from '@supabase/supabase-js'
    import { Database } from './database.types'

    const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
    const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

    if (!supabaseUrl || !supabaseAnonKey) {
      throw new Error('Missing Supabase environment variables')
    }

    export const supabase = createClient<Database>(supabaseUrl, supabaseAnonKey)
    ```

    Create src/hooks/useAuth.ts implementing Pattern 2 from 01-RESEARCH.md (Session Management with Auth State Listener):

    Export useAuth hook that:
    1. Fetches existing session on mount via `supabase.auth.getSession()`
    2. Sets up `onAuthStateChange` listener for sign in/out events
    3. Returns `{ session, user, loading, signOut }` where:
       - session: Session | null
       - user: User | null (extracted from session)
       - loading: boolean (true until initial session check completes)
       - signOut: async function that calls `supabase.auth.signOut()`
    4. Cleans up subscription on unmount

    Follow the exact pattern from 01-RESEARCH.md Code Examples section to avoid Pitfall 4 (session not persisting across refresh).

    CRITICAL: Call getSession() BEFORE setting up onAuthStateChange listener, otherwise initial session won't be detected.
  </action>

  </verify>
  <done>
    useAuth hook exists, exports session/user/loading/signOut, calls getSession on mount, sets up auth listener, cleans up on unmount, no TypeScript errors.
  </done>

    const handleSubmit = async (e: React.FormEvent) => {
      e.preventDefault()
      setLoading(true)

        email,
